Developing Identifiers for Heritage Collections

This resource describes the different facets of developing persistent identifiers (PIDs) in the context of Heritage Collections. It is aimed at senior technical stakeholders within galleries, libraries, archives and museums to assist with developing requirements and advocacy for new services.

Before you begin...

Please consult the Read Me section before using this resource.

Aspects for Developing PIDs for Collections

Augment with existing IDs

PIDs such as Wikidata IDs, ISNIs, Getty URIs, DOIs, Gazetteer URIs and IDs from other collections can be added to collection management systems. These can be included for related collection items, an identifier for a collection item itself where it is represented in another identifier scheme and authority files.

Assign IDs which could be persistent

Identifiers have a potentially machine readable format which could be expressed in a globally unique way via a DOI prefix, URI namespace or other.

Commit to persisting IDs

A PID can be presented as a link which resolves to a publicly accessible web page containing metadata about the item or concept. Minimum metadata is required. Metadata has an open license to enable reuse.

Humanly resolvable PIDs

A PID can be presented as a link which resolves to a publicly accessible web page containing metadata about the item or concept. Minimum metadata is required. Metadata has an open license to enable reuse.

Machine readable PIDs

PIDs and metadata are made available in a machine readable format and via an API or SPARQL or OAI-PMH endpoint. Minimum metadata is required. Metadata has an open license to enable reuse.

Globally resolvable PIDs

The identifiers are created under the auspices of an external identifier scheme such as ARKs, DOIs etc. The identifiers are resolvable via this scheme, rather than an institution specific domain (web address). This introduces contractual requirements for persistence and maintenance of access to resources.

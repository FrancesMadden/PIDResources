  1. Supporting Materials

Getting Started

The aspects are arranged in a general progression but a single institution would not have to implement all of these aspects.

Dependencies refer to other parts of this resource only. They mention the other aspects which would have had to have been implemented in order to implement that aspect.

Compatible PID types are included where the aspect describes an externally facing aspect of implementation.

All aspects require a baseline collections system to be in place.

The Baseline

Identifiers assigned within the metadata for physical items and/or digital objects. The identifiers are not globally unique, machine readable or resolvable. This is where we assume most users of this resource will be and the various facets build on this.

Explanation

This is what most organisations use now. These could be accession numbers, archive reference number, barcodes, internal database identifiers. The identifiers might be used in different systems but there is no computer interaction between systems using the identifier.

Benefits

You can use them to find items within your collection, however often they only work across one type of collection, e.g. you cannot search the museum collection management system using the archive reference number. Internal reporting is possible.

Feedback

We value your feedback on this resource. Any suggestions on the content or additional examples would be welcome.

To submit feedback, you can raise an issue via Github (requires an account) or you can use the Feedback button in the bottom right of each screen.

    Examples

    Any museum with a collection management system such Adlib, TMS, Emu etc. will meet this baseline level. Archives using a management system such as Calm. Libraries using a system such as Aleph or Alma.
